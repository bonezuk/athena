Track Data and Album Cover

Black Omega supports a number of different metadata formats used to embed track information within the contents of its file. This information includes name of the track, its album, position within the album, performing artist, composer, genre and album cover. This information is employed to both organise and display track information within the playlist.

The player understands the following metadata formats

  1. ID3Tag version 1 and 2.
  2. iTunes M4A
  3. Vorbis comments
  4. Flac comments and CUE sheets.

It also understands when a given audio file is split into multiple tracks.

  1. Flac with Cuesheets representing the contents of a CD it was created from.
  2. M4B audiobook could contain multiple chapters.

Black Omega is able to support these file formats and creates the appropriate tracks in the playlist.

Although there are a good number of track metadata editors available it can sometimes prove to be awkward, if not impossible, to embed the album cover image into the associated tracks. Black Omega gets around this limitation or simplifies the need to specifically edit each track by looking for an image file in the same directory as the albumâ€™s audio tracks. If more than one image file exists Black Omega will choose the image file whose name is the most similar to the name of the given album.
